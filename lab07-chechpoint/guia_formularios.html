01

Reset global e custom properties

style.css — primeiras linhas

Antes de qualquer regra visual, duas coisas precisam acontecer: o browser precisa ser "zerado" para um comportamento previsível, e a paleta de cores/tamanhos precisa virar variável. Isso evita que você repita valores como #003366 em quinze lugares diferentes — quando a cor muda, você altera um único ponto.

CSS para escrever

style.css
/* ===========================
   RESET + CUSTOM PROPERTIES
   =========================== */

*, *::before, *::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

:root {
  /* Cores */
  --cor-principal: #003366;
  --cor-destaque:  #FFCC00;
  --cor-texto:     #1a1a2e;
  --cor-fundo:     #f8fafc;
  --cor-superficie:#ffffff;
  --cor-borda:     #c8d0dc;
  --cor-erro:      #c62828;
  --cor-sucesso:   #2e7d32;
  --cor-muted:     #6b7280;

  /* Tipografia */
  --fonte: 'Segoe UI', system-ui, sans-serif;

  /* Geometria */
  --raio: 6px;
}
Por que box-sizing: border-box? Com content-box (padrão), um input com width: 100% e padding: 10px ultrapassa o container. Com border-box, padding e border são absorvidos pela largura declarada — o input fica exatamente no espaço esperado.
Por que custom properties no :root? Declaradas em :root, as variáveis ficam disponíveis em todo o documento. Elas são herdadas, podem ser sobrescritas por seletores filhos (dark mode!) e respondem a media queries — algo impossível com variáveis de pré-processadores estáticos.
Cuidado com o margin: 0 global O reset zera margin e padding de todos os elementos. Você vai precisar adicionar espaçamento de volta via margin-bottom em parágrafos e grupos de campo — de forma intencional, não acidental.

03

Fieldset e legend — agrupando campos

fieldset · legend

O <fieldset> tem uma borda padrão feia do browser e o <legend> uma tipografia inconsistente. A boa notícia: ambos são sobrescritos facilmente. O padrão de borda + legenda uppercase é usado em produtos como o GitHub e o Notion.

style.css
fieldset {
  border:         1px solid var(--cor-borda);
  border-radius:  var(--raio);
  padding:        1.1rem 1.25rem;
  margin-bottom:  1.5rem;
}
/* Remove a borda padrão do último
   fieldset antes dos botões */
fieldset:last-of-type {
  margin-bottom: 0;
}

legend {
  font-size:      .75rem;
  font-weight:    700;
  letter-spacing: .08em;
  text-transform: uppercase;
  color:          var(--cor-principal);
  padding:        0 .5rem;
}
Como o legend funciona no DOM O <legend> é um filho especial do fieldset — ele "flutua" para cima da borda superior. O padding: 0 .5rem cria o espaçamento que faz a borda "abrir" em torno do texto.

04

Label, grupo de campo e hint

label · .campo · .hint

Cada campo precisa de três camadas: o <label> descritivo, o controle em si, e opcionalmente um texto auxiliar (hint). O espaçamento vertical correto entre esses três elementos define o ritmo de leitura do formulário.

style.css
.campo {
  margin-bottom: 1rem;
}
.campo:last-child {
  margin-bottom: 0;
}

label {
  display:      block; /* ocupa linha inteira */
  font-size:    .88rem;
  font-weight:  600;
  color:        var(--cor-principal);
  margin-bottom:.3rem;
}
label .req {
  color: var(--cor-erro);
  margin-left: .1rem;
  font-weight: 700;
}

.hint {
  display:   block;
  font-size: .75rem;
  color:     var(--cor-muted);
  margin-top:.25rem;
}
Preview — estrutura do label
Conforme documento oficial
label display: block é obrigatório Sem display: block, o label fica inline e o input aparece ao lado dele na mesma linha, não embaixo. Leitores de tela ainda funcionam, mas o layout quebra visualmente.
05

Inputs de texto — text, email, tel, url, search

input[type=text] · :hover · :focus · :valid · :invalid

Estes cinco tipos compartilham a mesma aparência — o browser os diferencia internamente pelo teclado mobile e pela validação. Um único bloco CSS cobre todos, com pseudo-classes para os estados interativos.

CSS para escrever

style.css
input[type="text"],
input[type="email"],
input[type="tel"],
input[type="url"],
input[type="search"] {
  width:       100%;
  padding:     .55rem .85rem;
  border:      1.5px solid var(--cor-borda);
  border-radius:var(--raio);
  font-family: var(--fonte);
  font-size:   1rem;
  color:       var(--cor-texto);
  background:  var(--cor-superficie);
  transition:  border-color .2s ease,
               box-shadow .2s ease;
}

/* Estado hover — sinaliza interatividade */
input[type="text"]:hover,
input[type="email"]:hover /* ... */ {
  border-color: #9bb0cc;
}

/* Foco — WCAG 2.4.7 exige visibilidade */
input[type="text"]:focus,
input[type="email"]:focus /* ... */ {
  outline:      none;
  border-color: var(--cor-principal);
  box-shadow:   0 0 0 3px rgba(0,51,102,.15);
}

/* Validação visual nativa */
input:invalid:not(:placeholder-shown) {
  border-color: var(--cor-erro);
  box-shadow:   0 0 0 3px rgba(198,40,40,.12);
}
input:valid:not(:placeholder-shown) {
  border-color: var(--cor-sucesso);
}
Por que :not(:placeholder-shown)? Sem esse qualificador, um campo required vazio aparece imediatamente como :invalid — assim que a página carrega, antes do usuário digitar qualquer coisa. :not(:placeholder-shown) garante que o feedback de erro só apareça depois que o campo foi tocado.
box-shadow como anel de foco Usar box-shadow: 0 0 0 3px em vez de outline permite controle total de cor e tamanho, além de respeitar o border-radius do elemento. O outline: none é seguro aqui porque estamos fornecendo um substituto visível.

06

Textarea — área de texto livre

textarea · resize
style.css
textarea {
  width:       100%;
  padding:     .55rem .85rem;
  border:      1.5px solid var(--cor-borda);
  border-radius:var(--raio);
  font-family: var(--fonte); /* herda! */
  font-size:   1rem;
  line-height: 1.55;
  color:       var(--cor-texto);
  background:  var(--cor-superficie);
  min-height:  100px;
  resize:      vertical; /* só vertical */
  transition:  border-color .2s, box-shadow .2s;
}
textarea:focus {
  outline:      none;
  border-color: var(--cor-principal);
  box-shadow:   0 0 0 3px rgba(0,51,102,.15);
}
font-family não é herdado em textarea Por padrão, <textarea> usa uma fonte monoespaçada definida pelo browser. Você precisa declarar font-family: var(--fonte) explicitamente — a propriedade não herda automaticamente de body.
Preview
Redimensionável verticalmente pelo usuário
resize: vertical vs none resize: none bloqueia o redimensionamento — prejudica a usabilidade em textos longos. resize: vertical é o equilíbrio: o usuário pode aumentar a altura, mas não distorcer o layout expandindo horizontalmente.
07

Select — dropdown customizado

select · appearance · ::after

O <select> é um dos controles mais difíceis de estilizar — o browser renderiza sua lista de opções fora do contexto CSS. A técnica abaixo cobre a aparência do "disparador" (a caixa clicável), substituindo a seta nativa por uma customizada via pseudo-elemento no pai.

style.css
/* Wrapper para posicionar a seta customizada */
.select-wrap {
  position: relative;
}
.select-wrap::after {
  content:  '▾';
  position: absolute;
  right:    .85rem;
  top:      50%;
  transform:translateY(-50%);
  color:    var(--cor-principal);
  pointer-events: none; /* clique passa p/ o select */
  font-size:.85rem;
}

select {
  width:             100%;
  padding:           .55rem 2.5rem .55rem .85rem;
  border:            1.5px solid var(--cor-borda);
  border-radius:     var(--raio);
  font-family:       var(--fonte);
  font-size:         1rem;
  color:             var(--cor-texto);
  background:        var(--cor-superficie);
  appearance:        none; /* remove seta nativa */
  -webkit-appearance:none;
  cursor:            pointer;
  transition:        border-color .2s, box-shadow .2s;
}
select:focus {
  outline:      none;
  border-color: var(--cor-principal);
  box-shadow:   0 0 0 3px rgba(0,51,102,.15);
}
Preview
Limitação do select CSS não estiliza o <option> de forma consistente — o browser controla a lista drop-down. Para seletores totalmente customizados (cor por opção, ícones, busca), é necessário reconstruir o componente com JavaScript e ARIA. Para formulários simples, o nativo é a escolha certa.
08

Radio e Checkbox — pill interativo

accent-color · :has() · label wrapping

A técnica de "pill clicável" envolve o input dentro do label — o clique em qualquer parte da pílula seleciona a opção. A propriedade accent-color troca a cor do marcador nativo sem precisar reconstruí-lo do zero.

style.css
/* Grupo de opções em linha */
.radio-group,
.check-group {
  display:   flex;
  flex-wrap: wrap;
  gap:       .5rem;
  margin-top:.25rem;
}

/* Label como pill clicável */
.radio-group label,
.check-group label {
  display:      flex;
  align-items:  center;
  gap:          .45rem;
  padding:      .4rem .85rem;
  border:       1.5px solid var(--cor-borda);
  border-radius:20px;
  font-size:    .88rem;
  font-weight:  400; /* sobrescreve o bold do label */
  cursor:       pointer;
  user-select:  none;
  transition:   border-color .15s, background .15s;
}
.radio-group label:hover,
.check-group label:hover {
  border-color: var(--cor-principal);
  background:   #e8edf5;
}

/* Estado selecionado via :has() */
.radio-group label:has(input:checked),
.check-group label:has(input:checked) {
  border-color: var(--cor-principal);
  background:   #e8edf5;
  font-weight:  600;
}

/* Cor do marcador nativo */
input[type="radio"],
input[type="checkbox"] {
  accent-color: var(--cor-principal);
  width: 15px;
  height: 15px;
  cursor: pointer;
}
Preview — clique nas pílulas
:has() — o seletor de pai label:has(input:checked) seleciona o label que contém um input marcado. Disponível em todos os browsers modernos desde 2023. Antes, a mesma técnica exigia JavaScript ou o truque do + span com input escondido.

10

Color picker e upload de arquivo

input[type=color] · input[type=file] · label hack

O input[type="file"] é quase impossível de estilizar diretamente. A solução padrão de indústria: esconder o input e usar um <label> estilizado como botão. O input[type="color"] aceita CSS básico na caixa de cor, mas o picker nativo permanece controlado pelo browser.

style.css
/* Color picker */
input[type="color"] {
  width:         100%;
  height:        44px;
  padding:       2px 4px;
  border:        1.5px solid var(--cor-borda);
  border-radius: var(--raio);
  background:    var(--cor-superficie);
  cursor:        pointer;
  transition:    border-color .2s;
}
input[type="color"]:focus {
  outline:      none;
  border-color: var(--cor-principal);
  box-shadow:   0 0 0 3px rgba(0,51,102,.15);
}

/* Esconde o input nativo */
.file-input {
  position: absolute;
  opacity:  0;
  width:    0;
  height:   0;
}

/* Label estilizado como área de drop */
.file-label {
  display:       flex;
  align-items:   center;
  gap:           .6rem;
  border:        1.5px dashed var(--cor-borda);
  border-radius: var(--raio);
  padding:       .65rem .85rem;
  font-size:     .88rem;
  color:         var(--cor-muted);
  cursor:        pointer;
  transition:    border-color .2s, background .2s;
}
.file-label:hover {
  border-color: var(--cor-principal);
  background:   #e8edf5;
  color:        var(--cor-principal);
}
/* Foco acessível para navegação por teclado */
.file-input:focus-visible + .file-label {
  outline:      3px solid var(--cor-principal);
  outline-offset:2px;
}
Preview

Acessibilidade do file hidden O input escondido com opacity: 0 ainda recebe foco por teclado. Por isso use .file-input:focus-visible + .file-label para exibir o anel de foco no label — não no input invisível.

Date, datetime-local e Number

input[type=date] · input[type=number] · ::-webkit-inner-spin-button
style.css
input[type="date"],
input[type="datetime-local"],
input[type="month"],
input[type="time"] {
  width:         100%;
  padding:       .55rem .85rem;
  border:        1.5px solid var(--cor-borda);
  border-radius: var(--raio);
  font-family:   var(--fonte);
  font-size:     1rem;
  color:         var(--cor-texto);
  background:    var(--cor-superficie);
  transition:    border-color .2s, box-shadow .2s;
}
input[type="date"]:focus,
input[type="datetime-local"]:focus {
  outline:      none;
  border-color: var(--cor-principal);
  box-shadow:   0 0 0 3px rgba(0,51,102,.15);
}

input[type="number"] {
  width:         100%;
  padding:       .55rem .85rem;
  border:        1.5px solid var(--cor-borda);
  border-radius: var(--raio);
  font-family:   var(--fonte);
  font-size:     1rem;
  transition:    border-color .2s, box-shadow .2s;
}
/* Remove setas de incremento no Chrome */
input[type="number"]::-webkit-inner-spin-button,
input[type="number"]::-webkit-outer-spin-button {
  -webkit-appearance: none;
}
/* Remove seta no Firefox */
input[type="number"] {
  -moz-appearance: textfield;
}
Preview
Remover as setas do number As setas de incremento do type="number" ocupam espaço e o estilo delas varia muito entre browsers. Removê-las cria aparência consistente — o usuário ainda pode digitar o valor ou usar ↑↓ do teclado.
CEP, CPF e telefone: use type="text" Dados que parecem números mas não são quantidades matemáticas (zeros à esquerda, formatação com traço) devem usar type="text" com inputmode="numeric" e pattern.

Botões — primary, secondary e estados de foco

button · :hover · :active · :focus-visible · :disabled

O botão de envio é o elemento mais clicado do formulário — e o mais negligenciado em acessibilidade. Além dos estados hover e active, :focus-visible e :disabled são obrigatórios para um produto profissional.

style.css
.btn-row {
  display:    flex;
  gap:        .75rem;
  flex-wrap:  wrap;
  margin-top: 1.5rem;
}

/* Botão primário */
.btn-primary {
  flex:         1;
  padding:      .75rem 1.5rem;
  background:   var(--cor-principal);
  color:        #ffffff;
  border:       none;
  border-radius:var(--raio);
  font-family:  var(--fonte);
  font-size:    1rem;
  font-weight:  700;
  cursor:       pointer;
  transition:   background .2s, transform .12s;
  min-height:   44px; /* WCAG 2.5.5 */
}
.btn-primary:hover  { background: #00509e; }
.btn-primary:active { transform:  scale(.97); }
.btn-primary:focus-visible {
  outline:       3px solid var(--cor-destaque);
  outline-offset:3px;
}
.btn-primary:disabled {
  background: #9bb0cc;
  cursor:     not-allowed;
  opacity:    .65;
}

/* Botão secundário (outline) */
.btn-secondary {
  padding:      .75rem 1.25rem;
  background:   transparent;
  color:        var(--cor-principal);
  border:       1.5px solid var(--cor-principal);
  border-radius:var(--raio);
  font-family:  var(--fonte);
  font-size:    .95rem;
  cursor:       pointer;
  min-height:   44px;
  transition:   background .2s, color .2s;
}
.btn-secondary:hover {
  background: var(--cor-principal);
  color:      #fff;
}

  outline:       3px solid var(--cor-principal);
  outline-offset:3px;
}
Preview — experimente Tab e Enter

Pressione Tab para navegar — observe o anel de foco amarelo

:focus-visible vs :focus :focus é disparado também ao clicar com mouse — exibe o anel mesmo quando não é necessário. :focus-visible só exibe o anel quando o browser detecta navegação por teclado. Resultado: mouse não vê o anel, teclado sempre vê.
min-height: 44px — WCAG 2.5.5 O critério Target Size (Nível AA no WCAG 2.2) exige área mínima de toque de 24×24px; 44px é o valor recomendado por Apple e Google para conforto em touchscreen.
